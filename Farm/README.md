# Farm

## Описание
Проект представляет собой консольный симулятор фермерского хозяйства на платформе .NET 8.0. Архитектура приложения построена на взаимодействии различных сущностей (работники, животные, техника, поля) через абстракции и конфигурационные файлы.

## Функциональность

### 1. Животноводство (Animals)
Реализована иерархия классов на базе абстрактного класса `Animal` и интерфейса `IAnimal`.
* **Конфигурация:** Характеристики животных (здоровье, голод, продуктивность, возрастные лимиты) задаются через класс `AnimalConfig`.
* **Жизненный цикл:** Животные стареют, потребляют пищу, производят продукты, загрязняют место пребывания и могут умереть при низком уровне здоровья.
* **Виды:** Реализованы `Cow`, `Chicken`, `Pig`, `Rabbit`, `Sheep`, `Goat`, `Duck`. Каждое животное имеет уникальное поведение (например, `DigForWorms` у куриц, `RollInMud` у свиней) и производит определенный продукт (молоко, яйца, шерсть, мясо).

### 2. Агрокультура и Поля (Fields)
Поля реализованы как наследники абстрактного класса `Field` (производный от `Place`).
* **Типы полей:** `CornField`, `PotatoField`, `WheatField`, `CabbageField`, `FruitField`.
* **Управление:** Каждое поле имеет свои методы ухода (полив `Irrigate`, удобрение `Fertilize`, прополка `Weed` и т.д.), влияющие на `SoilCareLevel` и итоговую продуктивность.
* **Механика:** Реализована посадка семян со склада (`TryPlantFromWarehouse`), расчет урожайности и сбор продукции.

### 3. Техника (Machines)
Система машин разделена на самоходную технику и навесное оборудование, реализует интерфейс `IMachine`.
* **Самоходная техника:**  `Tractor`: Может использовать навесное оборудование.
    * `Harvester`: Комбайн, требует квалифицированного водителя (`EquipmentOperator`).
* **Навесное оборудование:** `Plow` (плуг), `Seeder` (сеялка), `CropSprayer` (опрыскиватель). Подключаются к трактору через интерфейс `IAttachableMachine`.
* **Управление:** Техника требует назначения водителя, включения двигателя (`TurnOn`) и может перемещаться между локациями.

### 4. Персонал (Employees)
Работники наследуются от класса `Employee` и имеют уровни квалификации (`Intern` — `Senior`), влияющие на зарплату и эффективность.
* **Роли:**
    * `Farmer`: Ухаживает за животными, собирает продукты. Реализована логика забоя животных (свиньи, кролики) на мясо.
    * `EquipmentOperator`: Управляет техникой (`SitInMachine`), выполняет работы на полях.
    * `FieldWorker`: Выполняет ручную работу или помогает оператору техники (связка через `ConnectToOperator`).
    * `SalesManager`: Занимается продажей продукции со склада, генерируя выручку.
    * `Accountant`: Рассчитывает зарплаты на основе выполненной работы и управляет повышением квалификации персонала (продвижение по `EmployeeLevel`).

### 5. Экономика и Продукты
* **Продукция:** Базовый класс `Product` отслеживает количество, свежесть и уровень повреждений. Цена зависит от свежести. 
* **Обработка:** Продукты проходят постобработку после сбора (например, `Pasteurize` для молока, `VacuumPack` для мяса), что влияет на их характеристики.
* **Склад (Warehouse):** Централизованное хранилище для продуктов и семян. Позволяет хранить урожай и выдавать семена для посадки.

## Тестирование
Проект `FarmTests` использует фреймворк xUnit.
* **Покрытие:**
    * **Animals:** Проверка создания, кормления, смены локаций, специфических действий (звуки, уникальные методы).
    * **Employees:** Тесты для всех ролей, проверка начисления зарплат, взаимодействия оператора и полевого работника, ограничения на управление техникой.
    * **Fields:** Проверка агротехнических операций (удобрение, полив) и механики посадки.
    * **Machines:** Тесты прицепления/отцепления оборудования, проверки прав доступа водителей.
    * **Products:** Проверка влияния повреждений на свежесть и цену.
* **Утилиты:** Используется `TestUtils` для перехвата консольного вывода и верификации сообщений симуляции.
